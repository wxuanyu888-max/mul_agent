# 需求文档：Memory 更新和记忆管理

## 需求背景

当前项目已有 Memory 索引系统（位于 `src/memory/`），支持：
- **向量搜索**：基于 embedding 的语义搜索
- **全文搜索**：FTS 全文检索
- **混合搜索**：结合向量和关键词
- **文件监听**：自动同步文件系统变化

但目前的功能主要集中在**文件内容的索引和搜索**，缺少**用户主动管理记忆**的能力。

## 初步理解

根据当前代码分析，可能涉及以下功能方向：

### 方向 1：手动记忆管理
- 手动添加记忆（不只是索引文件）
- 更新/删除已有记忆
- 记忆分类（短期、长期、交接）

### 方向 2：记忆生命周期管理
- 记忆过期机制
- 记忆优先级/权重
- 记忆关联（知识图谱）

### 方向 3：记忆检索增强
- 多维度检索（时间、来源、类型）
- 记忆摘要生成
- 相关记忆推荐
