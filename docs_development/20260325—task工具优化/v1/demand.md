# 需求文档：Task 工具优化

## 需求背景

Agent 在执行复杂任务时，不愿意使用 task 工具派发给子智能体，导致任务串行执行效率低下。

## 问题分析

1. **交接文档要求过于繁琐**：task 工具描述中要求"每次调用前必须使用 memory 工具写入 handover 类型的记忆"
2. **缺乏使用指导**：没有明确说明何时应该使用 task 工具
3. **Agent 心理门槛高**：繁琐的要求让 Agent 倾向于自己执行而不是派发子任务

## 期望结果

1. Agent 愿意主动使用 task 工具派发子任务
2. 复杂任务能够并行处理，提升执行效率
3. 降低 Agent 使用 task 工具的心理门槛

## 约束条件

- 不能破坏现有的子智能体功能
- 需要保持 task 工具的核心功能不变
- 修改后需要通过类型检查
