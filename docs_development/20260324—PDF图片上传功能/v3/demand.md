# 需求文档：PDF和图片上传功能 V3 - 文件解析状态显示

## 需求背景

用户反馈：
1. 前端上传文件后没有交互反馈，应该有解析状态显示
2. 想了解不同文件的处理逻辑和 Agent 的交互设计

## V2 问题回顾

V2 已实现：
- ✅ 支持上传 Markdown (.md) 文件
- ❌ 没有文件解析状态反馈
- ❌ 没有文件内容提取处理
- ❌ Agent 无法访问文件内容

## 文件处理逻辑分析

### 项目已有的解析能力

项目中已集成 `pdf-parse` 和 `mammoth` 库，用于 memory 系统的文件解析：

**现有能力** (`src/memory/parser.ts`):
- `parseDocument()` 支持：PDF、DOCX、DOC、TXT、MD
- `indexFile()` 自动将内容分块、向量化、存入 SQLite

### 不同文件的处理方案

| 文件类型 | 处理方式 | 原因 |
|----------|----------|------|
| **图片** | MiniMax API vision | LLM 直接识别图片 URL |
| **PDF** | pdf-parse 提取文字 → 向量存储 | 项目已有 pdf-parse |
| **Markdown** | 读取内容 → 向量存储 | 纯文本直接处理 |
| **文本** | 读取内容 → 向量存储 | 纯文本直接处理 |

### 完整处理流程

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  前端上传   │────▶│  后端存储   │────▶│  返回文件ID │
│  文件       │     │  uploads/  │     │  + 元数据   │
└─────────────┘     └─────────────┘     └─────────────┘
                                            │
                    ┌───────────────────────┘
                    ▼
        ┌─────────────────────────┐
        │  调用解析 API            │
        │  /files/:id/extract     │
        └─────────────────────────┘
                    │
        ┌───────────┴───────────┐
        ▼                       ▼
┌───────────────┐     ┌──────────────────┐
│ 图片          │     │ PDF/MD/TXT        │
│ 传 URL 给LLM  │     │ 提取内容          │
│ (直接识别)    │     │ ↓                 │
└───────────────┘     │ 存入向量数据库     │
                     │ ↓                 │
                     │ 返回文本内容给Agent │
                     └──────────────────┘
```

### Agent 交互设计

**消息格式传给 Agent:**
```
用户消息: "帮我分析这个文档"

附件内容:
[附件: document.pdf]
- 提取的文字内容...

Agent 收到完整上下文，包含提取的文件内容
```

## 详细需求

### 需求 1: 前端文件解析状态显示

1. **上传时显示解析状态**
   - 上传前：显示"待上传"
   - 上传中：显示"上传中..." + 进度
   - 上传成功：显示"解析中..." + 状态图标
   - 解析成功：显示"✓ 已解析" + 内容摘要
   - 解析失败：显示"✗ 解析失败" + 错误原因

2. **文件预览增强**
   - 添加解析状态指示器
   - 支持查看提取的文本内容（可选）
   - 显示文件大小和类型信息

### 需求 2: 后端文件内容提取

1. **图片文件处理**
   - 传 URL 给 LLM，MiniMax API 支持 vision
   - 不需要额外处理

2. **PDF 文件处理**
   - 使用 pdf-parse 提取文本
   - 将内容存入向量数据库（MemoryIndexManager）
   - 返回提取的文本内容

3. **Markdown/文本文件处理**
   - 读取文件内容
   - 将内容存入向量数据库
   - 返回文本内容

4. **内容传递到 Agent**
   - 提取的文本内容放入消息中
   - Agent prompt 说明附件内容处理方式

### 需求 3: 向量数据库存储

1. **使用现有 MemoryIndexManager**
   - 文件解析后调用 indexFile 存入向量库
   - 可通过关键词搜索查找文件内容

2. **存储标识**
   - 文件来源标记为 "upload"
   - 方便后续检索

## 详细描述

### 前端交互设计

```
┌──────────────────────────────────────────────────┐
│  📄 test.md                        2.5 KB    [X] │
│  ⏳ 解析中...                                        │
└──────────────────────────────────────────────────┘

解析成功时：
┌──────────────────────────────────────────────────┐
│  📄 test.md                        2.5 KB    [X] │
│  ✅ 已解析 · 提取 150 字                           │
└──────────────────────────────────────────────────┘

解析失败时：
┌──────────────────────────────────────────────────┐
│  📄 test.md                        2.5 KB    [X] │
│  ❌ 解析失败: 无法读取文件内容                      │
└──────────────────────────────────────────────────┘
```

### 后端处理流程

1. **文件上传** (`POST /api/v1/files/upload`)
   - 保存文件到 storage/uploads/
   -返回文件元数据

2. **内容提取** (新增 endpoint)
   -`GET /api/v1/files/:fileId/extract`
   -根据文件类型调用不同的提取器
   -返回提取的文本内容
   -自动存入向量数据库

3. **消息发送时**
   -前端上传文件 → 获取文件 ID
   -调用内容提取 API → 获取文本内容
   -将文本内容放入消息体发送给 Agent

## 期望结果

1. 用户上传文件后可以看到解析状态
2. 不同文件类型有不同的处理逻辑
3. Agent 能够访问文件内容进行理解和回复
4. PDF/MD 内容存入向量数据库，支持后续检索
5. 支持图片 vision、PDF 文本提取、Markdown 读取
