# 需求文档：PDF和图片上传功能 V2 - Markdown支持

## 需求背景

用户反馈：当前前端仍不能上传 markdown 文件，需要添加对 `.md` 文件的支持。

## V1 问题回顾

V1 已实现：
- ✅ PDF 上传和显示
- ✅ 图片上传和显示
- ❌ Markdown 文件不支持

## 详细描述

### 功能需求

1. **支持上传 Markdown 文件**
   - 添加 `.md` 文件类型到允许列表
   - 前端文件选择器支持 .md 文件
   - 前端和后端都允许 .md 类型

2. **文件预览和显示**
   - 在输入预览区显示 markdown 文件图标
   - 在消息中显示已发送的 markdown 文件

### 期望结果

1. 用户可以点击上传按钮选择 .md 文件
2. 用户可以拖拽 .md 文件到输入区域
3. 上传前显示文件预览（图标 + 文件名）
4. 发送消息时文件一并发送
5. 聊天记录中显示已发送的 .md 文件

### 约束条件

1. 文件大小限制：单个文件最大 10MB（与现有限制一致）
2. 仅支持纯文本 markdown 文件
3. 后端存储，前端展示文件信息

## 确认的方案

### 后端处理
- 添加 `text/markdown` 和 `.md` 到允许类型
- 文件存储到 `storage/uploads/` 目录

### 前端交互
- 按钮上传：支持选择 .md 文件
- 拖拽上传：支持拖拽 .md 文件
- 预览：显示 markdown 文件图标
- 消息显示：显示文件名和文件图标
