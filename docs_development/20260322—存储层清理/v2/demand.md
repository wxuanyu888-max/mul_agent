## 关键问题

### 1. BaseStorageManager 要不要用？

**现状**: 创建了基类但没有被使用

**方案 A: 让 Manager 继承基类** (推荐)
- 减少代码重复
- 统一缓存/刷新逻辑
- 风险：重构可能破坏现有功能

**方案 B: 保持现状**
- 不动现有代码
- 基类保留供未来使用
- 短期风险低，但代码重复问题持续

### 2. 日志轮转要不要现在做？

**现状**: 配置存在但未实现
```typescript
maxFileSize: 10 * 1024 * 1024, // 10MB - 未使用
maxFiles: 10,                    // 未使用
```

**方案 A: 现在实现**
- 完成 v1 未竟之事
- 避免日志撑爆磁盘

**方案 B: 暂时不做**
- 工作量大，收益有限
- 当前日志量不大

### 3. TS 错误优先级？

项目有多个 TS 错误，但有些是"死代码"（引用了不存在的模块）。

**方案 A: 全部修复**
- 工作量大，可能涉及删除代码

**方案 B: 只修复阻塞编译的错误**
- 优先保证 `pnpm build` 能过

**方案 C: 标记为 TODO，先不做**
- 这些是历史遗留问题
- 不影响核心功能
