# 工具返回值优化

## 需求背景

在 Agent 循环中，当工具调用失败时，现有的实现可能会抛出异常导致 Agent 停止运行，无法完成用户任务。用户期望 Agent 能够持续运行，即使遇到错误也能继续尝试其他方法。

## 详细描述

### 问题分析

1. **LLM 调用失败**: 网络问题或 API 错误时，LLM 调用会抛出异常导致 Agent 停止
2. **工具执行异常**: 部分工具内部存在 `throw` 语句，异常会向上传播导致 Agent 停止
3. **MCP 工具连接失败**: Browser MCP 工具在连接失败时会抛出异常

### 期望结果

1. **LLM 调用**: 添加重试机制（最多 3 次），失败后添加错误消息到上下文，让 Agent 继续尝试
2. **工具执行**: 所有工具都应该返回错误结果，而不是抛出异常
3. **Agent 持续运行**: 任何错误都不应该导致 Agent 终止

### 约束条件

- 不能破坏现有的工具调用逻辑
- 错误信息需要返回给 Agent，让它能够做出响应
- 保持向后兼容

## 验收标准

- [ ] LLM 调用失败时 Agent 继续运行（最多 3 次重试）
- [ ] 工具执行失败时返回错误结果，不抛出异常
- [ ] 类型检查通过
- [ ] Agent 能够处理错误并尝试其他方法
