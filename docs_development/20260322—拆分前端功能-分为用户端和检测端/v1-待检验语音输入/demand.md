# 需求文档：语音功能集成到Chat面板

> 状态：**待检验**
> 优先级：P0

## 1. 需求背景

原来的语音功能（VoiceChat）是一个独立的导航页面，用户体验割裂。按照"小步快走"的原则，先将语音功能集成到Chat面板中，提升用户体验。

## 2. 需求目标

### 2.1 交互设计
- 在Chat输入栏**左侧**添加一个小圆点语音按钮（🎤）
- 点击语音按钮，输入栏切换为**语音输入模式**
- 语音输入模式下：
  - 显示"正在聆听..."提示
  - 实时显示识别文字
  - 再次点击停止聆听
  - 识别完成后用户手动发送

### 2.2 视觉设计
```
普通模式：
[🎤] [输入框...............................] [发送]

语音模式：
[🔴] [正在聆听...已识别 X 字  ] ← 提示条在输入框顶部
    [===================输入框===================]
```

## 3. 验收标准

- [x] 语音按钮在输入栏左侧
- [x] 点击语音按钮进入聆听模式
- [x] 实时显示识别文字和字数统计
- [x] 再次点击停止聆听
- [x] 停止后识别结果保留在输入框，由用户手动发送
- [x] 语音功能使用浏览器 Web Speech API
- [x] **界面元素对齐**
- [x] MiniMax TTS 语音合成功能
