# 需求文档：Workflow增强 - Agent可视化监控

> 状态：需求讨论
> 版本：v2

## 1. 需求背景

当前 Workflow 页面已经实现了基本的可视化功能，但用户希望能更直观地看到 Agent 的运行情况。

## 2. 当前已实现功能

| 功能 | 状态 | 说明 |
|------|------|------|
| 节点显示 | ✅ | User、Core Brain、Agent Team |
| 状态显示 | ⚠️ | 只显示 running，缺少细分 |
| 交互边 | ✅ | 显示交互类型 |
| Agent详情弹窗 | ✅ | AgentDetailsModal |
| 交互历史弹窗 | ✅ | InteractionHistoryModal |
| 当前任务 | ✅ | currentWork 显示在节点上 |

## 4. 需求目标

### 4.1 Agent 状态细分（优先级：高）

增强节点状态显示：

| 状态 | 颜色 | 说明 |
|------|------|------|
| idle | 灰色 | 空闲 |
| planning | 黄色 | 思考/规划中（闪烁） |
| executing | 绿色 | 执行工具中（闪烁） |
| waiting | 蓝色 | 等待响应 |
| completed | 蓝色 | 完成 |
| error | 红色 | 出错 |

### 4.2 工具调用详细显示（优先级：高）

- 显示工具类型图标
- 显示工具名称（如果可用）
- 执行中的工具高亮显示

### 4.3 节点显示增强（优先级：中）

- 显示当前执行的具体任务
- 显示工具调用进度
- 节点点击弹出更详细的 Agent 面板

### 4.4 实时性优化（优先级：中）

- 缩短刷新间隔（从 3 秒改为 1-2 秒）
- 添加状态变化动画

## 5. 技术方案

### 5.1 前端增强（本期）

基于现有 API 数据增强 UI：

1. **状态颜色增强**
   - 根据现有数据做更细致的颜色区分
   - 添加动画效果让状态更直观

2. **节点信息增强**
   - 显示更多当前任务信息
   - 添加进度条

3. **交互反馈**
   - 悬停显示详情
   - 点击显示更完整的历史

### 5.2 后端配合（后续）

如果前端增强不够，可能需要：
- 后端返回更详细的状态
- 添加 WebSocket 实时推送

## 6. 验收标准

- [ ] Agent 节点显示更细致的状态（即使只有 running 也加动画）
- [ ] 工具调用显示类型和名称
- [ ] 节点显示当前执行任务
- [ ] 状态刷新更及时（1-2秒）
- [ ] 悬停/点击交互更流畅
