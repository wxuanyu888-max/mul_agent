# Workflow 画板重设计需求

> 文档位置：`docs_development/20260325—Workflow画板重设计/demand.md`

## 1. 需求背景

当前 WorkflowCanvas 使用 @xyflow/react 实现静态的节点-边图结构，agents 以固定位置显示，交互通过连线展示。

**问题：**
- 视觉效果静态、缺乏活力
- 无法直观展示 agents 的"活跃度"和"空闲状态"
- 交互形式单一

**期望：**
- 更生动的可视化效果，展示 agents 的动态行为
- 直观区分"忙碌"和"空闲"状态
- 通过位置和连线变化反映真实的工作状态

---

## 2. 详细需求

### 2.1 画板边界

| 属性 | 描述 |
|------|------|
| **边界类型** | 矩形边界，agents 不能超出 |
| **边界样式** | 圆角矩形边框，带柔和阴影或渐变效果 |
| **画板尺寸** | 响应式，占满容器 |
| **边界内边距** | agents 游走时与边界保持一定距离 |

### 2.2 空闲状态（自由游走）

**行为：**
- User 和所有 agents 在画板内随机游走
- 移动方式：平滑的随机漫步动画（类似粒子漂浮）
- 每个实体有独立的移动速度和方向变化
- 实体之间不碰撞，可以重叠

**视觉表现：**
- User 节点保持可见
- Agents 以半透明或"idle"样式显示
- 不显示 agent 之间的连接线
- Core Brain 可以保持在中心位置或轻微浮动

### 2.3 任务对话时（吸附到中心）

**触发条件：**
- 有新的 interaction（type: chat/bash/memory 等）
- Agent 被调用执行任务

**行为：**
- 被调用的 agent 从自由位置平滑动画到画板中心区域
- 如果是 delegation/collaboration，相关多个 agents 都被吸引到中心
- 从 User/Core Brain 到被调用 agent 创建连接线
- 连接线可以有方向箭头

**视觉表现：**
- 被吸附的 agent 恢复正常不透明度
- Agent 节点放大或高亮（表示活跃）
- 连接线显示，带动画效果（脉冲/流动）
- 可以显示任务类型标签（如 "chat", "bash"）

### 2.4 释放机制

**触发条件：**
- 任务完成（status: completed/success）
- 一段时间没有新交互（超时阈值可配置）

**行为：**
- Agent 从中心平滑动画回到自由游走状态
- 连接线淡出消失
- Agent 逐渐变为半透明（idle 样式）

**超时时间：**
- 默认 30 秒无交互后释放
- 可配置

---

## 3. 技术约束

### 3.1 技术选型

- 继续使用 React（已有的技术栈）
- 可以考虑：
  - **纯 CSS/Framer Motion**：动画简单，不需要引入新库
  - **@xyflow/react**：已有，可以扩展其动画能力
  - **Canvas/SVG**：如果需要更复杂的粒子效果

### 3.2 性能考虑

- 动画应该使用 `requestAnimationFrame` 或 CSS 动画
- 避免使用 setInterval 进行位置更新
- 节点数量：支持 10-20 个 agents 流畅动画

---

## 4. 交互设计

### 4.1 点击行为

| 元素 | 点击行为 |
|------|----------|
| 空闲 agent | 显示 agent 详情弹窗（已有功能） |
| 活跃 agent | 显示 agent 详情弹窗 + 当前任务 |
| 连接线 | 显示 interaction 历史（已有功能） |

### 4.2 拖拽行为

- 空闲状态：可以拖拽 agent 到任意位置，agent 保持该位置
- 活跃状态：agent 不可拖拽，被锁定在中心

---

## 5. 视觉风格

### 5.1 颜色方案

| 状态 | Agent 样式 |
|------|-----------|
| 空闲 | 半透明（opacity: 0.5-0.7），灰色边框 |
| 活跃 | 全不透明度，彩色边框，发光效果 |
| 执行中 | 脉冲动画，边框高亮 |

### 5.2 连接线

| 类型 | 样式 |
|------|------|
| User → Agent | 实线，带方向箭头 |
| Core Brain → Agent | 虚线，紫色系 |
| Agent ↔ Agent | 点线，协作颜色 |

---

## 6. 需求确认（已确认）

| 项目 | 确认内容 |
|------|----------|
| 动画风格 | 随机漂浮（像水面上的叶子） |
| Core Brain | 一样自由游走，不固定在中心 |
| 画板背景 | 白色，无网格/点阵 |
| 释放时间 | 5分钟无交互后释放 |
| Agent 点击 | 保持显示/编辑功能不变 |
| Controls | 暂时不开发缩放/拖拽等功能 |

## 7. 关键实现细节

### 7.1 边界定义

- 画板为矩形区域，agents 不能超出边界
- 边界内边距：50px（防止贴边）
- 当 agent 飘到边界时会反弹（改变方向）

### 7.2 自由游走算法

```
每个 agent 有：
- position: {x, y}
- velocity: {vx, vy}  // 速度向量
- 随机每帧改变速度方向（小幅度的）
- 边界碰撞检测：超出边界时反弹
```

### 7.3 吸附逻辑

```
当有新的 interaction 时：
1. 找到 source 和 target agent
2. 将它们标记为 "active" 状态
3. 平滑动画到画布中心附近（不要求完全重合，允许一定分散）
4. 创建连接线
5. 开始 5 分钟计时
```

### 7.4 释放逻辑

```
当 5 分钟无新交互时：
1. 将所有 active agents 标记为 "idle"
2. 淡出连接线
3. agents 恢复自由游走状态
```

### 7.5 状态定义

| 状态 | 视觉表现 |
|------|----------|
| idle | 半透明，浮动画布上 |
| active | 全不透明度，在中心区域，显示连接线 |
| executing | 脉冲动画，边框高亮 |
