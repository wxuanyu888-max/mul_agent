# 需求文档：工具grep增强

## Issue
#2 - 工具的grep和group的补充（先做grep）

## 需求背景
现在的基本工具不是很全，探索功能不够。

## 确认的需求

### 1. 增强 grep 工具的上下文控制

**现有问题：**
- `exact` 模式：硬编码只返回匹配行前后各1行
- `semantic` 模式：上下文由向量搜索决定，我们无法控制

**解决方案：**
- 添加 `contextLines` 参数，让用户控制返回上下文行数
- 默认值保持兼容（1行）

### 2. JSON 文件关键信息提取

**需求：**
- 对 JSON 文件提取关键字段值
- 前端代码（.tsx, .jsx）不进行 JSON 提取处理

**待确认：**
- 具体要提取 JSON 中的哪些字段？
- 是提取所有字段值，还是按特定规则提取？

## 实施计划

1. 添加 `contextLines` 参数到 grep 工具
2. 修改 exactSearch 函数支持可配置上下文
3. 添加 JSON 文件处理逻辑（提取关键文字）
4. 前端代码跳过 JSON 提取
5. 编写测试

## 优先级
1. ✅ contextLines 参数（明确）
2. ⏳ JSON 提取（待确认细节）
3. ⏳ group 工具（待讨论）
