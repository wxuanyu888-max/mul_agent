# Cron 工具问题修复

## 需求背景

用户在使用 Cron 工具时，发现创建的定时任务默认会跑到下一天执行，而不是按预期立即开始。

## 问题描述

- 用户创建 `* * * * *`（每分钟执行）的定时任务
- 期望：任务应该在 1 分钟后首次执行
- 实际：任务被安排到下一天的同一时间（如当前 3:30，会安排到 3/26 16:00）

## 根因分析

`parseCronExpression` 函数对通配符 `*` 的处理存在逻辑错误：

1. **分钟为 `*` 时**：被错误地设为 0 分钟，然后判断时间已过就推到下一天
2. **小时为 `*` 时**：`date.setHours(date.getHours())` 没有任何作用
3. **未处理 `* * * * *`**：全通配符应该表示"每分钟"，但实现未正确处理

## 解决方案

修改 [cron-manager.ts](src/tools/system/cron-manager.ts) 中的 `parseCronExpression` 函数：

1. 当 `min === '*'` 时，直接返回 `now + 60000`（1分钟后）
2. 修正分钟处理逻辑：正确处理指定分钟的时间计算

## 验证

- `* * * * *` → 每分钟执行 ✓
- `30 14 * * *` → 每天 14:30 执行 ✓
- `0 9 * * 1-5` → 工作日 9:00 执行 ✓
