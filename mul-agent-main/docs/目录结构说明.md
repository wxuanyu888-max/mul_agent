# Mul-Agent 目录结构说明

## 存储架构

**所有存储都在 `wang/` 目录下**，不再使用 `storage/` 目录。

## wang/agent-team 目录规范

### 每个 Agent 只允许包含 5 个文件

| 文件名 | 用途 | 必需 |
|--------|------|------|
| `user.md` | Agent 角色、职责、能力定义 | 是 |
| `soul.md` | Agent 核心特质、行为模式、进化规则 | 是 |
| `skill.md` | Agent 技能、工具使用能力 | 是 |
| `memory.md` | Agent 记忆系统配置 | 是 |
| `logic.md` | Agent 逻辑规则、路由决策、工具权限 | 是 |

## wang/token_usage 目录

### 用途
存储所有 Agent 的 Token 使用统计。

### 文件结构
```
wang/token_usage/
```

## ConfigManager API

### 初始化
```python
from pathlib import Path
from mul_agent.brain.config_manager import ConfigManager

WANG_DIR = Path("/path/to/mul-agent/wang")
config_manager = ConfigManager(config_dir=WANG_DIR, wang_dir=WANG_DIR)
```

### 核心配置操作
```python
# 加载配置
config = config_manager.load(agent_id, "user")    # 加载 user.md
config = config_manager.load(agent_id, "soul")    # 加载 soul.md
config = config_manager.load(agent_id, "skill")   # 加载 skill.md
config = config_manager.load(agent_id, "memory")  # 加载 memory.md
config = config_manager.load(agent_id, "logic")   # 加载 logic.md
```

```python
# 加载 Token 使用统计
```

## API 路由

所有 API 路由都已更新为使用 `wang/` 目录：

| 路由 | 方法 | 描述 |
|------|------|------|
| `/api/v1/agents` | GET | 列出所有 Agent（从 wang/agent-team） |
| `/api/v1/agents/{agent_id}` | GET | 获取 Agent 详情 |
| `/api/v1/agents/{agent_id}/status` | GET | 获取 Agent 状态 |
| `/api/v1/token-usage` | GET | 获取所有 Agent 的 Token 使用统计 |
| `/api/v1/token-usage/{agent_id}` | GET | 获取指定 Agent 的 Token 使用统计 |
| `/api/v1/token-usage/{agent_id}/reset` | POST | 重置 Agent 的 Token 使用统计 |
| `/api/v1/info/agent-team` | GET | 获取 Agent 团队信息 |

## 迁移说明

### 从旧结构迁移

如果之前使用 `storage/` 目录，需要：

1. **移动 Agent 配置**
   ```bash
   # 移动 agent-team 配置
   cp -r storage/agent_config/* wang/agent-team/
   ```

2. **移动 Token 使用统计**
   ```bash
   # 移动 token_usage 文件
   cp storage/token_usage/*.json wang/token_usage/
   ```

3. **移动快照**
   ```bash
   # 移动快照
   cp storage/snapshots/* wang/snapshots/
   ```

4. **更新代码**
   - 所有 `STORAGE_DIR` 改为 `WANG_DIR`
   - 所有 `AGENT_CONFIG_DIR` 改为 `AGENT_TEAM_DIR`

### 代码更新要点

```python
# 旧代码
STORAGE_DIR = BASE_DIR / "storage"
AGENT_CONFIG_DIR = STORAGE_DIR / "agent_config"
config_manager = ConfigManager(config_dir=STORAGE_DIR, wang_dir=BASE_DIR / "wang")

# 新代码
WANG_DIR = BASE_DIR / "wang"
AGENT_TEAM_DIR = WANG_DIR / "agent-team"
config_manager = ConfigManager(config_dir=WANG_DIR, wang_dir=WANG_DIR)
```

## 最佳实践

1. **配置分离**
   - 核心配置（user/soul/skill/memory/logic）放在 `wang/agent-team/{agent_id}/`
   - Token 统计放在 `wang/token_usage/`
   - 项目文件放在 `wang/projects/`
   - 工作缓冲区放在 `wang/file-history/`

2. **模板使用**
   - 创建新 Agent 时使用 `.templates/` 中的模板
   - 保持配置一致性

3. **版本控制**
   - 所有配置文件包含 `version` 字段
   - 修改配置时更新版本号

4. **快照机制**
   - 每次保存配置前自动创建快照
   - 快照保存在 `wang/snapshots/`
