## 1️⃣ Token 使用优化

### 实现原理
通过 `_should_reflect()` 方法智能判断是否需要调用 LLM 进行深度反思。

### 判断规则
| 条件 | 是否反思 |
|------|----------|
| 失败/错误 | ✅ 是 |
| 关键操作（file_edit/memory） | ✅ 是 |
| 简单成功（bash/response <300 字符） | ❌ 否 |
| 关键 Agent 对话（coder/alice/bob） | ✅ 是 |

### 核心代码
```python
def _should_reflect(self, step, result) -> bool:
    # 失败必须反思
    if result.get("status") != "success":
        return True
    # 关键操作必须反思
    if step.get("route") in ["file_edit", "memory"]:
        return True
    # 简单成功跳过
    if len(str(result)) < 300:
        return False
    return True
```

## 7️⃣ 缓存优化

### 实现方法
- `_get_cache_key()` - 生成缓存键
- 缓存只读操作结果

### 缓存范围
| 路由 | 是否缓存 |
|------|----------|
| bash | ✅ 是 |
| glob | ✅ 是 |
| grep | ✅ 是 |
| file_edit | ❌ 否（写操作） |
| chat | ❌ 否（状态相关） |

### 核心代码
```python
def _get_cache_key(self, step):
    import hashlib
    key = f"{route}:{json.dumps(params, sort_keys=True)}"
    return hashlib.md5(key.encode()).hexdigest()
```
