/**
 * MulAgent 记忆系统重新设计方案 v3
 *
 * 核心理解：
 * 1. Short Memory = 会话加载（已存在）
 * 2. Vector Memory = 杂乱的、碎片化的信息（什么都能往里丢）
 * 3. Workspace = 只在 prompt 告知，不做工具
 */

# 整体架构

```
storage/
├── memory/                 # 向量记忆（杂乱的碎片箱）
│   ├── MEMORY.md          # 整洁的长期记忆（可选）
│   ├── notes/             # 各种杂乱的笔记
│   │   ├── xxx.md
│   │   ├── yyy.md
│   │   └── ...
│   └── daily/             # 每日记录
│       └── YYYY-MM-DD.md
└── workspace/             # 工作文件（只在 prompt 告知）
    ├── docs/
    ├── data/
    └── logs/
```

# Vector Memory (向量记忆)

## 定位

**杂乱的记忆碎片箱**：
- 什么都能往里丢
- 不需要整理、不需要分类
- 正因为杂乱，才需要向量搜索

## 什么内容进向量库？

| 内容类型 | 示例 | 为什么需要向量搜索 |
|---------|------|------------------|
| **零散想法** | "那个 API 好像有问题" | 难以分类，但以后可能相关 |
| **临时记录** | "用户提到了 xxx" | 片段信息，以后要搜索 |
| **不确定的** | "可能需要看看 xxx" | 不确定有没有用，先存着 |
| **各种片段** | 一句话、一个小发现 | 无法归类，但可能有用 |
| **用户提到的事** | "之前那个问题..." | 以前的相关信息 |

## 什么内容不应该进向量库？

| 内容 | 原因 |
|-----|------|
| 完整代码 | workspace 里直接读 |
| 文档 | workspace 里直接读 |
| 日志 | workspace 里直接读 |
| 整洁的决策 | 可以写 MEMORY.md（但也不强制） |

## 核心原则

```
用户说 "记住这个" → 丢进向量库
LLM 觉得可能有用 → 丢进向量库
不需要整理 → 直接写
不需要归类 → 直接写
反正乱 → 靠搜索找
```

# Workspace (工作区)

## 只在 Prompt 告知，不做工具

**Prompt 示例**：

```
## 工作区
位置: storage/workspace/
- docs/: 项目文档
- data/: 数据文件
- logs/: 日志文件

需要时直接读取。
```

**LLM 自己的决策**：
- 需要读代码 → 读 workspace
- 需要查日志 → 读 workspace
- 需要参考文档 → 读 workspace

---

# 写入流程

## LLM 的决策（非常简单）

```
收到信息
    │
    ▼
想记住吗？
    │
    ├─ 否 → 不写
    │
    ▼
写
    │
    ▼
直接写到 storage/memory/notes/xxx.md
    │
    ▼
向量系统自动索引
（不需要决定写哪里、怎么分类）
```

---

# 搜索流程

## 什么时候搜索？

- 用户问"之前那个..."
- LLM 想确认"之前有没有提到过..."
- 任何需要查找"之前"信息的时候

---

# System Prompt

```markdown
## 记忆系统

### 记忆位置
- storage/memory/notes/ - 各种笔记（会被自动索引搜索）
- storage/workspace/ - 工作文件（代码、文档、日志）

### 使用方式
- 想记住什么 → 直接写到 storage/memory/notes/ 下的任意 .md 文件
- 不需要整理，不需要归类
- 需要查之前的事 → 用 memory_search 搜索

### 工作区
- 位置: storage/workspace/
- docs/: 文档
- data/: 数据
- logs/: 日志
- 需要直接读取
```

---

# 总结

| 组件 | 作用 |
|-----|------|
| **notes/** | 杂乱的记忆碎片箱（向量搜索） |
| **daily/** | 每日记录 |
| **workspace/** | 原始文件（prompt 告知） |

**核心**：不要让 LLM 思考"这个应该放哪里"，直接写就对了，反正是杂乱的，靠搜索找。
