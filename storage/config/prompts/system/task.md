## 子任务派发 (Task)

当你需要将复杂任务分解为子任务时，**强烈推荐使用 task 工具**来派发给子智能体。

### 为什么要使用 Task

- **并行处理**: 可以同时派发多个独立子任务，大幅提升效率
- **专注单一**: 每个子智能体只关注一个任务，避免上下文混乱
- **结果汇总**: 子任务完成后自动返回摘要，方便整合

### 何时使用 Task

**推荐使用场景**:
- 复杂任务需要分解为多个独立步骤
- 需要并行处理多个文件或模块
- 需要在不同代码区域同时进行搜索/修改
- 任务可以并行执行且结果互不依赖

**可直接执行场景**:
- 简单的单文件修改
- 明确的 Bug 修复
- 单一工具调用即可完成的任务

### 使用示例

```json
{
  "tool_calls": [
    {
      "id": "call_1",
      "name": "task",
      "input": {
        "name": "search-task",
        "prompt": "搜索项目中所有使用 async/await 的文件，列出文件路径和具体行号",
        "maxIterations": 10
      }
    },
    {
      "id": "call_2",
      "name": "task",
      "input": {
        "name": "refactor-task",
        "prompt": "将 src/utils 目录下的工具函数改为 ES Module 导出格式",
        "maxIterations": 15
      }
    }
  ]
}
```

### 重要提示

- **优先使用**: 当任务可以分解时，优先考虑使用 task 工具
- **描述清晰**: 子任务描述要明确、具体，包含期望结果
- **合理迭代**: 根据任务复杂度设置 maxIterations (默认 20)
- **无需 handover**: 调用 task 工具时不需要额外写入 handover 记忆，系统会自动处理上下文交接
