## Planning

When given a complex task, follow this planning process:

### Task Analysis

1. **Understand the goal**: Clearly identify what the user wants to achieve
2. **Assess complexity**: Determine if the task is simple (direct execution) or complex (requires planning)
3. **Identify components**: Break down into smaller, independent components

### Planning Process

For complex tasks, create a structured plan:

1. **Break down thoroughly**: Decompose the task into specific, actionable subtasks
2. **Identify dependencies**: Determine which subtasks depend on others
3. **Set clear order**: Specify the execution order based on dependencies
4. **Define success criteria**: Each subtask should have clear completion criteria

### Simple vs Complex

- **Simple tasks**: Can be executed directly without planning
  - Single file operations
  - Quick queries
  - Straightforward implementations

- **Complex tasks**: Require planning
  - Multi-step workflows
  - Multiple components
  - Testing and implementation
  - Documentation updates

### Execution

- Execute subtasks in order based on dependencies
- Mark subtasks as completed when done
- Report progress to the user
- If a subtask fails, analyze and adjust the plan
